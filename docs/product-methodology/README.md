# 프로젝트 인덱스 — 프린팅 반팔 티셔츠 LLM 검색

> 제품 방법론 파이프라인 산출물 인덱스 · 최초작성 2026-07-21

## 한 줄 정의
사용자가 **말로 표현한 시각·감각 속성**("노란 프린팅 있는 흰 티", "산들산들한 느낌")으로 프린팅 반팔 티셔츠를 찾아주는 LLM 검색 서비스.

## 팀·제약
- 인원 4명 (개발자 = 나 1명, 나머지 3명 비개발)
- 기간 6주, 주 단위 스프린트
- 도구: Claude Code, OpenAI API, 네이버 오픈API, (선택) Supabase/Vercel/Notion/GA

## 핵심 결정 요약 (지금까지)
| # | 결정 | 근거 | 상태 |
|---|---|---|---|
| D1 | 카테고리 = 프린팅 반팔 티셔츠 1종 고정 | 스펙이 아닌 '느낌·그래픽'으로 고르는 물건 → LLM 검색 가치 큼 | 확정 |
| D2 | 핵심 문제 = 말로 표현한 시각·감각 속성으로 못 찾음 | 색×프린팅 조합, "산들산들함" 같은 무드/촉감 검색 부재 | 확정 |
| D3 | 데이터 뼈대 = 네이버 쇼핑 오픈API + 이미지 속성 추출 | API가 제목·이미지·가격·몰·카테고리 제공(무료, 일 25,000회) | 확정 |
| D4 | 데이터 소스 = 네이버 스마트스토어 생태계로 통일 | 상세·리뷰가 한 생태계라 처리 단순화 | 확정(단, 소재·리뷰 획득 난이도 높음 — data-feasibility 참고) |
| D5 | v1 검색축 = **시각 속성 핵심 / 소재는 추정 보조 / 리뷰는 소규모 샘플 실험** | 조사상 소재·리뷰가 가장 획득 어렵고 리뷰 크롤링은 법적·차단 리스크 | 확정 |
| D6 | 파이프라인 = 자동추출 → 사람 샘플검증 → 정확도 측정 → 신뢰도 확보 후 확장 | 비개발 3명이 검증·라벨링 담당(역할 적합) | 확정 |
| D7 | 첫 타겟 = **클라이머**(백프린팅 티에 진심). 대표 JTBD = "튀는 색 클라이밍화 코디용 티 찾기" | PM 본인=클라이머(직접관찰), 열정 니치, 기능성+그래픽 결합 pain | 확정 |
| D8 | 포지셔닝 = **흩어진 클라이밍 티를 모아 속성 조합(색×프린팅위치)으로 '말로' 찾고 '한눈에' 비교하는 발견 검색** | 문제=부재가 아니라 파편화. SPIRI7과 '시맨틱 조합 검색'으로 차별 | 확정 |
| D9 | 팀 역할 = 김홍교(개발총괄)·홍상호(카탈로그·라벨기준)·신유정(이미지속성·검증)·라진우(감각·기능성 오너) | 자동추출↔사람검증 루프에 대칭 배치. 라진우는 리뷰 크롤링 대신 텍스트 감각속성+샘플실험 | 확정 |
| D10 | 카탈로그 = 소규모 먼저 → 정확도 검증 후 2000 확장 | 6주/1인 현실성 + 품질 우선 | 확정 |
| D11 | **실행 구조 = 3×2주 Build-Measure-Learn 린 루프. Amplitude 계측. 프로젝트 핵심 = "데이터로 의사결정" 증명** | 완성도 아닌 린 루프 도는 PM 서사가 부트캠프에 강함 | 확정 |
| D12 | Loop1 = 최소 자동추출(**비전 LLM 단일 호출→JSON 속성**) + 카탈로그 50~100 + 검색 + Amplitude 배포 | 2주 안에 가치+기술 동시 검증. OpenCV/OCR은 필요 시 Loop2 | 확정 |
