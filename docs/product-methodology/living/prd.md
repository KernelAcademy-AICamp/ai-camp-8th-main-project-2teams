# PRD — 클라이밍 프린팅 티 발견 검색 (가제)
> 유형: living(변경됨) · 2026-07-21 · 방법: 린 PRD
> 팀 단일 기준 문서. 상세는 각 living/foundation 문서 링크 참조. 결정 근거는 [README](../README.md) D1~D12.

## 1. 배경
클라이머는 튀는 색(형광·원색) 클라이밍화에 어울리는 **백프린팅 티**를 원하지만, 네이버·무신사 검색은 색을 옷 전체로만 인식해 "백프린팅만 노란/바탕 흰" 같은 **조합**을 표현하지 못한다. 결과는 대표 썸네일뿐이라 등판·소재·사이즈를 상품마다 상세페이지로 확인해야 하고, 공급도 인스타·암장·직구·중고로 흩어져 있다. → **PM 본인이 4시간 탐색 후 실패**(직접관찰). 클라이밍 인구 ~50만·성장세, 그래픽 티 문화·기능성 담론 확립(→ [problem-validation](../foundation/problem-validation.md)).

## 2. 목표 (SMART)
- 6주 안에 **3×2주 린 루프**로 MVP를 배포하고, **각 루프 종료 시 Amplitude 데이터+인터뷰 기반 의사결정 1건**을 문서화한다([iterate-log](iterate-log.md)).
- **Loop 1 종료 시**: 배포 URL 확보 + 속성 추출 정확도 베이스라인 + 검색 성공률 베이스라인 측정.
- **프로젝트 핵심 성과 = "데이터로 의사결정하는 과정"을 3루프로 입증.**

## 3. 성공 지표 → 상세 [metrics.md](metrics.md)
- **북극성**: 검색 세션당 "원하는 티 발견(찜/모의담기)" 비율 / 발견까지 시간
- **선행**: 검색 성공률·CTR·찜률·모의담기율·구매의향·탐색시간
- **가드레일**: no-result율·오검색 체감
- 부트캠프 규정: GMV·구매CVR·Add-to-Cart 직접개선 주장 금지 → 선행지표로 검증

## 4. 페르소나 → [customer-profile.md](customer-profile.md)
실내 볼더링/클라이밍 즐기는 20~30대. 장비·코디·아이덴티티에 신경. 원정 암장 티 수집 문화 참여. (첫 타겟=beachhead)

## 6. 요구사항
### P0 (Loop 1)
- **카탈로그**: 네이버 쇼핑API로 클라이밍 프린팅 반팔티 50~100개 수집 + 큐레이션
- **속성 추출**: 상품 이미지 → **비전 LLM 단일 호출 → JSON**. 스키마 아래 §7.
- **검증 루프**: 자동추출 → 사람 샘플 검증(속성별 정확도) → 임계 미달 속성은 검색 노출 제외/플래그
- **자연어 검색**: 쿼리 → LLM 파싱(속성화) → 속성 필터·랭킹
- **결과 카드**: 등판 이미지 · 추출 색칩(바탕/프린팅) · 핵심속성(핏·소재·사이즈유무) 미리 노출
- **찜(모의 저장)** + **Amplitude 계측**(§metrics 이벤트) + Vercel 배포
### P1 (Loop 2~3)
- 카탈로그 확장(200~500) · 추출 정밀화(색 분리 OpenCV 보조/OCR) · 신발색 매칭 훅 · 필터 고도화
### P2 (v2+)
- 리뷰 본격 · 비주얼 유사검색 · 대화형 재질문 · 인접 카테고리(튀는 장비색 코디: 러닝·아웃도어)

## 7. 데이터·기술
- **속성 스키마**: 바탕색 / 프린팅색 / 프린팅위치(앞·뒤·전체) / 그래픽유형(레터링·캐릭터·로고·패턴) / 핏(오버·레귤러·슬림) / 소재·기능성(면·폴리·기능성; 추정 플래그) / 사이즈(유무·프리) / 가격 / 몰링크·이미지
- **아키텍처(안)**: 네이버 쇼핑API 수집 → 이미지 비전LLM 추출 → 속성 DB(Supabase) → 검색(쿼리 LLM 파싱→속성 랭킹) → 프론트(Next.js + shadcn) → Vercel 배포 → Amplitude 로그
- **데이터 제약**(→ [data-feasibility](../foundation/data-feasibility.md)): 네이버 API는 제목·이미지·가격·몰·카테고리만(소재·리뷰 없음, 일 25,000회, 키워드당 1,000개 상한). 소재·리뷰는 API 부재 → 소재는 이미지/텍스트 추정, 리뷰는 소규모 샘플 실험만.

## 11. 릴리즈·실험 계획
Loop 1(W1-2) 가치검증+최소추출 → Loop 2(W3-4) 데이터가 가리키는 곳 투자 → Loop 3(W5-6) 정밀화·확장·마감. 매 루프 Amplitude+인터뷰 → 의사결정([iterate-log](iterate-log.md)). 최종 산출물: PRD·배포URL·랜딩·인터뷰·인사이트 보고서·발표자료.

## 12. 팀 역할
김홍교(개발총괄·파이프라인·검색·앱·배포) · 홍상호(카탈로그·라벨 기준) · 신유정(이미지 속성·검증) · 라진우(감각·기능성 속성·인터뷰).
