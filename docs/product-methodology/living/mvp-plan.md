# MVP 계획 (MVP Plan) — 3×2주 린 루프
> 유형: living · 2026-07-21(개정) · 방법: Build-Measure-Learn 루프 ×3
> **이 프로젝트의 핵심 = "데이터로 의사결정하는 과정"을 보여주는 것.** 완성도 자랑이 아니라 매 루프 끝에 데이터→결정이 나오는 걸 증명한다.

## 한 줄
말로 조합 검색하고(색×프린팅위치·핏·기능성) 한눈에 비교하는 **클라이밍 프린팅 반팔티 발견 검색**. 2주 루프로 만들고-재고-배우며 3회 반복.

## 전체 구조 (6주 = 2주 × 3)
```
Loop 1 (W1-2) 가치검증 + 최소 자동추출 → 배포 → Amplitude+인터뷰 → 의사결정
Loop 2 (W3-4) Loop1 데이터가 가리키는 곳에 투자        → 측정 → 의사결정
Loop 3 (W5-6) 정밀화·전환·확장 + 최종 산출물          → 측정 → 회고
```
매 루프 종료 시 [iterate-log.md](iterate-log.md)에 회고·배운것·피봇/지속 결정 기록.

---

## Loop 1 (W1-2) — "이 검색 방식이 쓸만한가?"
**목표 가설**: H1(추출 정확도 임계 달성) · H2(자연어 조합 검색이 탐색을 실제로 줄인다)
**최소 자동추출**: 네이버 API로 클라이밍 프린팅티 **50~100개** 수집 → 상품 이미지를 **비전 LLM 단일 호출→JSON**(바탕색/프린팅색/프린팅위치/그래픽유형/핏/소재추정)으로 속성화 → 사람 샘플 검증(속성별 정확도) → 임계 미달 속성은 검색서 숨김.
**빌드**: 자연어 쿼리→LLM 파싱→속성 랭킹 · 결과 카드(등판·색칩·핵심속성) · 찜 · **Amplitude 계측** · Vercel 배포.
**측정**: 검색 사용·no-result율·결과 클릭·찜·세션시간(→[metrics.md](metrics.md)) + 클라이머 인터뷰 3~5.
**의사결정 게이트**: 정확도 임계 통과? 검색이 실제로 원하는 걸 찾아주나? → Loop 2 투자처 결정.

| 팀원 | Loop 1 |
|---|---|
| 김홍교 | API 수집 + 비전LLM 추출 + 검색/파싱 + 프론트(Next+shadcn) + Amplitude + 배포 |
| 홍상호 | 속성 스키마 + 라벨 기준 + 카탈로그 50~100 큐레이션 |
| 신유정 | 시각속성(색·프린팅·그래픽) 샘플 정확도 검증 |
| 라진우 | 소재·기능성·느낌 태그 검증 + 클라이머 인터뷰 진행 |

## Loop 2 (W3-4) — "데이터가 가리키는 곳에 투자"
**입력 = Loop1 데이터.** 예: no-result 많음 → **카탈로그 확장(200~500)**; 특정 속성 오검색 → **추출 개선**(색 분리에 OpenCV 보조/세그멘테이션·OCR 추가); 쿼리 실패 → **파싱 개선**; 클릭은 되나 찜 안 됨 → **카드/필터 개선** or **신발색 매칭 훅(P1)**.
**측정·인터뷰 반복 → 의사결정 게이트.**

## Loop 3 (W5-6) — "정밀화·전환·확장 + 마감"
전환(찜·구매의향) 개선 · (신뢰도 되면)2000 확장 시도 · 감각/리뷰 소규모 실험 · **최종 인사이트 보고서(3루프 데이터→의사결정 서사)** · 발표자료 · PRD 최종 · 랜딩.

## 기능 스코프
- **P0(Loop1)**: 최소 자동추출 파이프라인 · 속성 스키마 · 검증 루프 · 자연어 검색 · 결과 카드 · 찜 · Amplitude
- **P1(Loop2~3)**: 카탈로그 확장 · 추출 정밀화 · 신발색 매칭 훅 · 필터 고도화
- **P2(v2+)**: 리뷰 본격 · 비주얼 유사검색 · 대화형 재질문 · 인접 카테고리(튀는 장비색 코디)

## "불편할 정도로" 줄인 것
- 실결제·실장바구니 없음 → 찜/모의 담기 (부트캠프 규정: GMV·CVR 직접개선 주장 금지, 선행지표로 검증)
- 리뷰 전면검색 없음(크롤링 리스크) → 소규모 실험만
- 카테고리 1개 · Loop1 카탈로그 50~100(→ 데이터 보고 확장)

## 검증할 가설 (상위 3)
- **H1(기술)**: 색 바탕/프린팅 분리 등 속성 추출이 신뢰할 정확도로 된다 → Loop1
- **H2(가치)**: 자연어 조합 쿼리로 탐색시간이 극적으로 준다(4시간+실패 → 수분)
- **H3(수요)**: 클라이머가 이 방식으로 실제 발견·저장한다

## 미해결
- H1 정확도 임계 수치(Loop1 결과 보고 확정)
- "클라이밍 관련" 카탈로그 선정 기준(브랜드·키워드·암장 굿즈) — 홍상호 Loop1 초반
- 비전LLM 추출 단가 vs 카탈로그 규모(비용 상한) — Loop2 확장 전 점검
