- **O1. 색×프린팅위치 '조합'을 검색으로 표현 못 함** ("백프린팅만 노란/바탕 흰"을 필터·검색어로 못 짬) — *가장 큼*
- **O2. 검색결과가 대표 썸네일뿐** → 등판·프린팅 보려 상세페이지 일일이 진입
- **O3. 사이즈·소재·기능성·질감을 상품마다 확인**해야 함(속성 미구조화)

## 실험 (Experiments)
- S1: 색 **바탕/프린팅 분리 추출** 정확도 PoC(비전 LLM 주, OpenCV 보조) — 샘플 30~50개 사람 검증 → 정확도 임계 판정 (**H1**)
- S1: 자연어 조합 쿼리 10개로 "원하는 결과 상위 노출" 성공률 측정 (**H2**)
- S2/S3: 카드만 보고 상세 진입 없이 후보 3개 고르기 태스크 성공률·소요시간
- S5: 매칭 훅은 P1 — 색 이론(보색/톤) 규칙 + LLM

## 선택한 가지 (지금 파는 것)
**O1+O2+O3을 v1 핵심**으로. S1(속성 시맨틱 검색)+S2(속성 카드)+S3(필터). O4는 카탈로그 수집으로 부분 해결. **O5는 라진우가 텍스트(제목·상세)에서 추정 + 리뷰 소규모 실험**으로 v1 보조, 본격은 v2. S5(매칭 훅)는 P1.
**이유**: JTBD가 "조합으로 빠르게 걸러 한눈에 비교"라 속성 검색+카드가 정확히 정조준. 데이터 구조화 노력이 그대로 자산화되고 지표 측정이 쉬움.
